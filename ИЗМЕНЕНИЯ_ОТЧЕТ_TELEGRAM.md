# Изменения в отчете Telegram (/day)

## Что добавлено

В метод `get_day_report_text` добавлена детализация по символам с покупками и продажами за день.

### Новые возможности:

1. **Детализация по каждому символу** с операциями за день:
   - Количество покупок и продаж
   - Общее количество акций и сумма
   - Время и цена каждой покупки/продажи
   - Реализованный P/L по символу (если были продажи)
   - Средняя цена входа (если позиция открыта)

2. **Формат вывода**:
   ```
   *Детализация по символам:*
   
   *PLTRUB_TOM*:
     Покупок: 2 (10 шт, 64,849.75 RUB)
     Продаж: 1 (10 шт, 66,659.90 RUB)
     Покупки: 07:48:12 @ 6486.98, 07:53:44 @ 6482.97
     Продажи: 11:42:35 @ 6665.99
     Реализованный P/L: ✅ 1,810.15 RUB (+2.79%)
   ```

3. **Автоматический расчет P/L**:
   - Используется метод average-cost для расчета реализованного P/L
   - Учитываются все покупки и продажи за день
   - Показывается процент прибыли/убытка

## Технические детали

- Используются данные из audit-лога (события `trade`)
- Символы нормализуются через `_canon_symbol()` для единообразия
- Расчет P/L выполняется по методу FIFO (first-in-first-out) через average-cost
- Детализация показывается только для символов с операциями за день

## Тестирование

Для тестирования можно использовать:
```bash
python test_day_report.py
```

Или запросить отчет через Telegram:
```
/day 2026-01-23
```

## Совместимость

- Изменения обратно совместимы
- Если нет сделок за день, детализация не показывается
- Все существующие метрики остаются без изменений
